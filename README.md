# ETL Pipeline для аналитики данных

## Описание проекта

Проект представляет собой ETL-пайплайн для загрузки данных из различных источников в аналитическое хранилище данных.

В рамках проекта реализованы:

* извлечение данных из файлов различных форматов (CSV, JSON, Excel, XML);
* очистка и валидация данных;
* контроль качества данных (Data Quality);
* загрузка данных в PostgreSQL;
* выполнение аналитических SQL-запросов;
* логирование ошибок обработки данных.

### Контроль качества данных

На этапе трансформации выполняются проверки данных на корректность и полноту.

Контролируются следующие типы ошибок:

* дублирующиеся записи;
* некорректные email-адреса;
* некорректные номера телефонов;
* отсутствующие обязательные поля;
* ошибки форматов дат;
* некорректные суммы платежей;
* нарушения ссылочной целостности.

Все обнаруженные ошибки фиксируются в журнале качества данных для последующего анализа.

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/fedinaa/project.git
cd project
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Создание базы данных

```bash
psql -U postgres -c "CREATE DATABASE mosmusic;"
```

### 4. Создание структуры хранилища

```bash
psql -U postgres -d mosmusic -f ddl/schema.sql
```

### 5. Настройка подключения к БД

В файле `src/config.py`:

```python
DB_CONFIG = {
    "host": "localhost",
    "port": "5432",
    "database": "mosmusic",
    "user": "postgres",
    "password": "postgres"
}
```

### 7. Запуск ETL-процесса

```bash
cd src
python main.py
```

### 8. Выполнение аналитических запросов

```bash
psql -U postgres -d mosmusic -f sql/analytics.sql
```
